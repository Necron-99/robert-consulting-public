Understanding GitOps for Infrastructure

GitOps applies DevOps best practices to infrastructure management:

  • Git as Source of Truth: All infrastructure changes are committed to Git
  • Automated Deployments: Changes are automatically applied when merged
  • Observability: Full audit trail of all infrastructure changes
  • Rollback Capability: Easy reversion to previous states

Consistency: Same deployment process across all environments
Security: All changes go through code review and approval
Auditability: Complete history of infrastructure changes
Collaboration: Team members can review and approve changes

GitHub Actions Workflow: Plan Phase

Create a workflow that runs `terraform plan` on every pull request:

# .github/workflows/terraform-plan.yml
name: 'Terraform Plan'

on:
  pull_request:
    branches: [ main ]
    paths:
      - 'environments/**'
      - 'modules/**'
      - '.github/workflows/terraform-plan.yml'

env:
  TF_VERSION: '1.6.0'
  TF_IN_AUTOMATION: true

jobs:
  plan:
    name: 'Terraform Plan'
    runs-on: ubuntu-latest
    
    strategy:
      matrix:
        environment: [dev, staging, production]
    
    steps:
    - name: Checkout
      uses: actions/checkout@v4
    
    - name: Setup Terraform
      uses: hashicorp/setup-terraform@v3
      with:
        terraform_version: ${{ env.TF_VERSION }}
    
    - name: Terraform Format Check
      run: terraform fmt -check -recursive
      working-directory: environments/${{ matrix.environment }}
    
    - name: Terraform Init
      run: terraform init
      working-directory: environments/${{ matrix.environment }}
      env:
        TF_CLI_ARGS: '-no-color'
    
    - name: Terraform Validate
      run: terraform validate
      working-directory: environments/${{ matrix.environment }}
    
    - name: Terraform Plan
      run: terraform plan -out=tfplan
      working-directory: environments/${{ matrix.environment }}
      env:
        TF_VAR_environment: ${{ matrix.environment }}
    
    - name: Comment Plan on PR
      uses: actions/github-script@v7
      with:
        script: |
          const fs = require('fs');
          const plan = fs.readFileSync('environments/${{ matrix.environment }}/tfplan', 'utf8');
          
          github.rest.issues.createComment({
            issue_number: context.issue.number,
            owner: context.repo.owner,
            repo: context.repo.repo,
            body: `## Terraform Plan - ${{ matrix.environment }}
            
            \`\`\`terraform
            ${plan}
            \`\`\`
            `
          });

GitHub Actions Workflow: Apply Phase

Create a workflow that applies changes after approval:

# .github/workflows/terraform-apply.yml
name: 'Terraform Apply'

on:
  push:
    branches: [ main ]
    paths:
      - 'environments/**'
      - 'modules/**'

env:
  TF_VERSION: '1.6.0'
  TF_IN_AUTOMATION: true

jobs:
  apply:
    name: 'Terraform Apply'
    runs-on: ubuntu-latest
    environment: ${{ matrix.environment }}
    
    strategy:
      matrix:
        environment: [dev, staging, production]
    
    steps:
    - name: Checkout
      uses: actions/checkout@v4
    
    - name: Setup Terraform
      uses: hashicorp/setup-terraform@v3
      with:
        terraform_version: ${{ env.TF_VERSION }}
    
    - name: Terraform Init
      run: terraform init
      working-directory: environments/${{ matrix.environment }}
      env:
        TF_CLI_ARGS: '-no-color'
    
    - name: Terraform Apply
      run: terraform apply -auto-approve
      working-directory: environments/${{ matrix.environment }}
      env:
        TF_VAR_environment: ${{ matrix.environment }}
    
    - name: Update Deployment Status
      uses: actions/github-script@v7
      with:
        script: |
          github.rest.repos.createCommitStatus({
            owner: context.repo.owner,
            repo: context.repo.repo,
            sha: context.sha,
            state: 'success',
            target_url: `${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}`,
            description: 'Terraform apply completed successfully',
            context: `terraform-apply-${{ matrix.environment }}`
          });

Security Scanning Integration

Add security scanning to your GitOps pipeline:

# .github/workflows/security-scan.yml
name: 'Security Scan'

on:
  pull_request:
    branches: [ main ]
    paths:
      - 'environments/**'
      - 'modules/**'

jobs:
  security-scan:
    name: 'Security Scan'
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout
      uses: actions/checkout@v4
    
    - name: Run Checkov
      uses: bridgecrewio/checkov-action@master
      with:
        directory: environments/
        framework: terraform
        output_format: sarif
        output_file_path: checkov-results.sarif
    
    - name: Upload Checkov results
      uses: github/codeql-action/upload-sarif@v2
      with:
        sarif_file: checkov-results.sarif
    
    - name: Run Trivy
      uses: aquasecurity/trivy-action@master
      with:
        scan-type: 'config'
        scan-ref: 'environments/'
        format: 'sarif'
        output: 'trivy-results.sarif'
    
    - name: Upload Trivy results
      uses: github/codeql-action/upload-sarif@v2
      with:
        sarif_file: trivy-results.sarif

State Management and Locking

Configure remote state with proper locking:

# environments/production/backend.tf
terraform {
  backend "s3" {
    bucket         = "my-company-terraform-state"
    key            = "production/terraform.tfstate"
    region         = "us-east-1"
    dynamodb_table = "terraform-state-lock"
    encrypt        = true
  }
  
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }
}

# DynamoDB table for state locking
resource "aws_dynamodb_table" "terraform_state_lock" {
  name           = "terraform-state-lock"
  billing_mode   = "PAY_PER_REQUEST"
  hash_key       = "LockID"
  
  attribute {
    name = "LockID"
    type = "S"
  }
  
  tags = {
    Name        = "Terraform State Lock"
    Environment = "shared"
  }
}

Rollback Strategy

Implement automated rollback capabilities:

# .github/workflows/terraform-rollback.yml
name: 'Terraform Rollback'

on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Environment to rollback'
        required: true
        default: 'staging'
        type: choice
        options:
        - dev
        - staging
        - production
      commit_sha:
        description: 'Commit SHA to rollback to'
        required: true
        type: string

jobs:
  rollback:
    name: 'Terraform Rollback'
    runs-on: ubuntu-latest
    environment: ${{ github.event.inputs.environment }}
    
    steps:
    - name: Checkout specific commit
      uses: actions/checkout@v4
      with:
        ref: ${{ github.event.inputs.commit_sha }}
    
    - name: Setup Terraform
      uses: hashicorp/setup-terraform@v3
      with:
        terraform_version: '1.6.0'
    
    - name: Terraform Init
      run: terraform init
      working-directory: environments/${{ github.event.inputs.environment }}
    
    - name: Terraform Apply
      run: terraform apply -auto-approve
      working-directory: environments/${{ github.event.inputs.environment }}
    
    - name: Notify Rollback Complete
      uses: actions/github-script@v7
      with:
        script: |
          github.rest.issues.createComment({
            issue_number: context.issue.number,
            owner: context.repo.owner,
            repo: context.repo.repo,
            body: `## Rollback Complete
            Environment: ${{ github.event.inputs.environment }}
            Rolled back to: ${{ github.event.inputs.commit_sha }}
            `
          });

Monitoring and Alerting

Set up monitoring for your GitOps pipeline:

# .github/workflows/terraform-monitoring.yml
name: 'Terraform Monitoring'

on:
  workflow_run:
    workflows: ["Terraform Apply"]
    types: [completed]

jobs:
  monitor:
    name: 'Monitor Deployment'
    runs-on: ubuntu-latest
    if: ${{ github.event.workflow_run.conclusion == 'failure' }}
    
    steps:
    - name: Send Failure Alert
      uses: actions/github-script@v7
      with:
        script: |
          // Send alert to Slack or email
          const payload = {
            text: `üö® Terraform deployment failed for ${{ github.event.workflow_run.head_branch }}`,
            blocks: [
              {
                type: "section",
                text: {
                  type: "mrkdwn",
                  text: `*Environment:* ${{ github.event.workflow_run.head_branch }}\n*Commit:* ${{ github.event.workflow_run.head_sha }}\n*Workflow:* ${{ github.event.workflow_run.html_url }}`
                }
              }
            ]
          };
          
          // Send to webhook (configure in repository secrets)
          await fetch(process.env.SLACK_WEBHOOK_URL, {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify(payload)
          });

Best Practices for GitOps
